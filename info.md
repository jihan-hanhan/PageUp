# PageUp - GitHub 文档管理工具

一款轻量级的跨平台文档管理工具，用于在 GitHub 仓库中存储和管理文档。

## 功能特性

### 1. 文档管理

| 功能 | 说明 |
|------|------|
| 创建文档 | 支持创建 TXT 和 Markdown 格式文档 |
| 编辑文档 | 实时编辑文档内容，支持 Markdown 语法 |
| 文档预览 | Markdown 文档支持实时预览 |
| 文件重命名 | 支持重命名已有文档 |
| 本地删除 | 删除本地列表中的文档 |

### 2. GitHub 集成

| 功能 | 说明 |
|------|------|
| PAT 登录 | 使用 Personal Access Token 登录 GitHub |
| 仓库连接 | 支持连接任意 GitHub 仓库（owner/repo 格式） |
| 文档存储 | 文档统一存储于仓库的 `docs/` 目录下 |
| Save & Push | Save 保存本地草稿，Push 推送至 GitHub |
| Push All | 一键推送所有文档变更 |
| 冲突处理 | 重命名文件后 Push 会自动删除旧文件 |

### 3. 用户体验

| 功能 | 说明 |
|------|------|
| 仓库历史 | 自动保存仓库连接历史，支持快速切换 |
| 历史管理 | 支持删除单条历史记录 |
| 深色模式 | 支持明暗主题切换 |
| 响应式设计 | 适配桌面端和移动端 |
| 移动端抽屉 | 侧边栏支持滑动手势展开/收起 |

### 4. 数据持久化

| 功能 | 说明 |
|------|------|
| PAT 存储 | 登录后自动保存 PAT，下次启动自动登录 |
| 登出清理 | 点击登出时清除本地 PAT |
| 本地草稿 | 文档修改后自动保存为本地草稿 |

## 技术栈

- **框架**: Tauri 2.x
- **前端**: Vue 3 + TypeScript + Vite
- **后端**: Rust
- **API**: GitHub REST API
- **构建**: GitHub Actions

## 支持平台

- ✅ Windows
- ✅ Android
- ⏳ macOS
- ⏳ iOS

## 使用说明

1. **登录**: 输入 GitHub Personal Access Token（需要 `repo` 权限）
2. **连接仓库**: 输入 `owner/repo` 格式的仓库地址
3. **创建文档**: 点击新建按钮，输入文件名和内容
4. **保存**: 点击 Save 按钮保存本地草稿
5. **推送**: 点击 Push 按钮推送至 GitHub

## 注意事项

- PAT 需要 `repo` 权限才能操作仓库
- 文档统一存储于仓库的 `docs/` 目录
- 第一次 Push 需要仓库权限，否则会提示权限不足
